# Web-Fetch 核心功能规划

## 项目概述

Web-Fetch 是一个面向 AI 的网页数据采集服务，将任意网站转换为 LLM 可用的结构化数据。

## 核心功能 (已完成 ✅)

### 1. 网页抓取 (Scrape) ✅
- 单 URL 转换为 Markdown / HTML / JSON
- 支持代理访问国外网站
- 输出格式：markdown, html, links

### 2. 代理支持 ✅
- 自动使用 HTTP_PROXY 环境变量
- 默认代理: http://127.0.0.1:7890
- 支持访问 BBC/Reuters 等国外网站

### 3. 格式转换 ✅
- HTML 转 Markdown
- 自动去除 script/style 标签
- 保留链接、图片等重要元素

### 4. 链接提取 ✅
- 自动提取页面所有 HTTP 链接
- 自动去重

## 待开发功能

### 1. 网站爬取 (Crawl)
- 从起始 URL 递归抓取整个网站
- 可配置最大深度 (maxDepth)
- 自动去重，避免重复抓取

### 2. 结构化提取 (Extract)
- 基于 Pydantic Schema 的结构化 AI 智能提取数据提取
-（无需预定义 Schema）
- 支持自定义提取规则

### 3. 搜索 (Search)
- Web 搜索能力
- 返回搜索结果 + 完整页面内容

### 4. Playwright 集成 (JS 渲染)
- 支持 JavaScript 渲染的动态页面
- 等待页面元素加载
- 截图功能

## API 设计

### POST /scrape
```json
{
  "url": "https://example.com",
  "formats": ["markdown", "html", "links"],
  "options": {
    "timeout": 30000
  }
}
```

### POST /crawl (规划中)
```json
{
  "url": "https://example.com",
  "maxDepth": 3,
  "concurrency": 5
}
```

## 输出格式

### Markdown
干净的 Markdown 内容，适合 LLM 处理

### JSON
结构化数据，支持 Schema 验证

### Metadata
- title
- description  
- sourceURL
- statusCode

## 技术实现

### 当前版本
- 使用 curl 作为 HTTP 客户端
- 支持代理转发
- 轻量级，无外部依赖

### 未来版本
- 集成 Playwright 支持 JS 渲染
- 添加缓存机制
- 支持分布式部署
