# Web-Fetch 核心功能规划

## 项目概述

Web-Fetch 是一个面向 AI 的网页数据采集服务，将任意网站转换为 LLM 可用的结构化数据。

## 核心功能 (全部完成 ✅)

### 1. 网页抓取 (Scrape) ✅
- 单 URL 转换为 Markdown / HTML / JSON
- 支持代理访问国外网站
- 输出格式：markdown, html, links

### 2. 网站爬取 (Crawl) ✅
- 从起始 URL 递归抓取整个网站
- 可配置最大深度 (maxDepth)
- 自动去重，避免重复抓取

### 3. 结构化提取 (Extract) ✅
- 自动提取标题、描述
- 自动提取链接列表
- 无需预定义 Schema

### 4. 搜索 (Search) ✅
- DuckDuckGo 搜索集成 (更稳定)
- 返回搜索结果列表
- 支持结果数量控制
- 支持获取页面详细内容

### 5. 代理支持 ✅
- 自动使用 HTTP_PROXY 环境变量
- 默认代理: http://127.0.0.1:7890
- 支持访问 BBC/Reuters 等国外网站

### 6. 缓存机制 ✅
- 本地文件缓存
- 默认 TTL: 1小时
- 支持配置
- 自动去重，减少重复请求

### 7. Playwright 支持 ✅
- JavaScript 动态页面渲染
- 支持需要 JS 的网站

## API 设计

### POST /scrape
```json
{
  "url": "https://example.com",
  "formats": ["markdown", "html", "links"],
  "options": {
    "timeout": 30000,
    "noCache": false
  }
}
```

### POST /search
```json
{
  "query": "AI news",
  "limit": 10,
  "options": {
    "includeContent": false,
    "noCache": false
  }
}
```

### POST /extract
```json
{
  "url": "https://example.com"
}
```

### GET /cache/status
获取缓存状态

```json
{
  "total": 10,
  "valid": 8,
  "expired": 2,
  "sizeMB": "0.05",
  "cacheDir": "/path/to/.cache"
}
```

### POST /cache/clear
清除缓存

```json
{
  "key": "specific_key"  // 可选，不填则清除所有
}
```

## 输出格式

### Markdown
干净的 Markdown 内容，适合 LLM 处理

### JSON
结构化数据

### Metadata
- title
- description  
- sourceURL
- fromCache (是否来自缓存)

## 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| PORT | 8080 | 服务端口 |
| HTTP_PROXY | http://127.0.0.1:7890 | HTTP 代理 |
| CACHE_ENABLED | true | 是否启用缓存 |
| CACHE_TTL | 3600 | 缓存过期时间(秒) |
| CACHE_DIR | .cache | 缓存目录 |
